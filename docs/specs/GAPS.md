## Structural Gaps (template sections missing from existing docs)

These are sections the Spec-Driven Design template set expects. The content
either never existed or is distributed across other files.

- [ ] **Testable Assertions (A-NNN) in all four PRDs** — `01-PRD-001-core.md`,
      `01-PRD-002-recovery.md`, `01-PRD-003-routines.md`,
      `01-PRD-004-sync.md` have Functional Requirements (FR-x.y) and Goals but
      no explicit `A-001`, `A-002` style testable assertions. The template
      structure calls for a dedicated "Testable Assertions" section per PRD.
      The invariants in `03-invariants.md` and FRs in each PRD can be the raw
      material, but assertion IDs need to be assigned and mapped.

- [ ] **Invariant-to-Assertion Mapping table** in `03-invariants.md` — the
      template calls for an explicit mapping of each invariant
      (H-1, C-2, REC-3, etc.) to at least one A-NNN assertion ID in a PRD.
      Cannot be populated until assertions exist.

- [ ] **Contract Surfaces table** in `05-erd.md` — the template expects a
      "Contract Surfaces" section listing contract file paths (e.g.,
      `domain/src/main/kotlin/.../HabitRepository.kt`) so sub-agents can
      reference them during parallel execution. Currently absent.

- [ ] **ADR-CANDIDATE markers in `04-architecture.md`** — the template's
      "Key Architecture Decisions" section asks for each decision to be
      tagged `ADR-CANDIDATE` or link to a ratified ADR. The existing
      architecture doc has decisions embedded without these markers. The one
      ratified ADR (`../adr-runtime-firebase-config.md`) is not back-referenced
      from the architecture doc's decisions list.

- [ ] **Cortex Appendix in `10-PLAN-001-kairos.md`** — the template's PLAN
      format expects a Cortex Appendix (Steps.md template, Contract Surface
      Index, Complexity Assessment). The existing implementation plan has the
      step-by-step structure and dependency graph but no Cortex-specific
      appendix. Safe to add when Cortex-driven execution begins.
